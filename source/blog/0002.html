htmlの構造

ここではhtmlの構造について、解説します。

htmlは空白・改行をあまり気にしない言語です。なので、全く改行せずに書くことができることはできます。分かりやすいところで改行するようにしましょう。

また、htmlは次のような構造をしています。

まず、「<!DOCTYPE html>」でこのファイルがhtmlだよとデバイスに教えます。

次に、「html」タグによって、ここからhtmlの記述を始めると宣言します。

「html」タグの中には実際に表示される「body」タグとページ全体に対する設定をする「head」タグがあります。
サンプルの「head」タグの中にはutf-8という文字コードで読みという指示を与えています。これがないと、文字ばけの原因になってしますので、必ず書くようにしてください。
タグの中には「head」のなかに書かないといけないものもあります。注意してください。

「body」タグの中にはさらに「header」と「main」と「footer」タグがあります。
「header」タグの中には一番の見出しなどを、「main」タグの中にはメインの内容を、「footer」タグ内にはサイトマップやサイトを運営する組織の情報を書いたりします。
どこに書いても基本的には同じなので、あまり気にしなくても大丈夫です。

なので、htmlの大きな構成はほとんど決まっています。↓よくある構成

<!DOCTYPE html>
<html>
	<head>
		<meta charset="utf-8">
	</head>

	<body>
		<header>
			<!--ここにheaderを書く-->
		</header>
	
		<main>
			<!--ここにmainを書く-->
		</main>
	
		<footer>
			<!--ここにfooterを書く-->
		<footer>
	
	</body>
</html>

大体のhtmlはこのような構造をしています。

綺麗なhtmlの書き方

「綺麗なhtml」とは？

「綺麗なhtml」とは表示のデザインが良いだけでなく、コードも無駄がなくメンテナンス性に優れていることです。コードが重複していたり、任意の部分の働きが一目でわからないと、綺麗とは言えないでしょう。

ここでは、綺麗なhtmlを作るコツを紹介したいと思います。

1、表示が綺麗なhtml

「デザインが良いもの」というよりも「見やすいもの」を目指しましょう。

表示が綺麗なhtmlは色のセレクトがいいことが多いです。特に背景色と文字の色の関係は大事です。基本的には小さい範囲には濃い色を、大きな範囲には薄い色を使いましょう。

また、空白は多めでいくのがおすすめです。cssのpaddingやmarginを多用しましょう。

2、無駄がないhtml

無駄がないようにするには、共通部分を重複しなように書くことが重要です。

footerのcssなど、共通のものは同じcssに書き、複数のhtmlに読ませるのは気軽にできるのでおすすめです。

また、html自体もJavaScriptを使うことによって、共通部分を外部ファイルに追い出すことができます。

これらの変更をするだけで、共通部分をいじれば全てのページに反映されるので、大幅にアップデートがしやすくなります。メンテナンス性も高くなりますね。

3、メンテナウンス性が高いhtml

何も、Webページ制作に限ったことではないですが、コメントを大量につけるようにしましょう。

また、「個別で指定したいということはないだろうな」と考え、divによるクラス付けを怠ってはいけません。
あとで、必要になるかもしれないので、全てのタグについてセレクタを使えるような状態にしておきましょう。
クラス名は「[親要素]-[子要素]」といちいちするのはおすすめです。要素自体の名前はアンダーバー("_")で繋ぎましょう。

html・cssは改行や空白の意味があまりない言語ではありますが、全く改行をしないとただの暗号になるので、改行・空白を空けるのは忘れずにやりましょう。

プログラミング全般において、解読不能のコードは挫折する原因になります。プロの方がむしろ、コメントを多用することを考えると、私たちのような素人にとってはメンテナウンス性はかなり重要であると思います。

例

ここでは、これまで紹介してきた工夫を全てこしらえた例を紹介します。

<!DOCTYPE html>
<html>
<head>	
	<!--文字コードと、検索された時の設定-->
	<meta charset="utf-8">
	<title></title>
	<!--検索したとき出る説明を設定-->
	<meta content="">
	
	<!--cssを読み込み-->
	<!--共通部分のcssを読み込み／共通部分のcssは相対参照に注意すること-->
	<link rel="stylesheet" href="css/base.css">
	<!--このページ独自のcssを読み込み-->
	<link rel="stylesheet" href="css/">

	<!--モバイル対応化のためのタグ-->
	<meta name="viewport" content="width=device-width, initial-scale=1.0">
</head>
<body>
	<div id="header"></div>
	<!--後のjsでここに、base_header.htmlを読み込み、追加する-->

	<main>
		<!--＊内容-->
		
		<!--内容終わり-->
	</main>

	<div id="footer"></div>
	<!--後のjsでここに、base_footer.htmlを読み込み、追加する-->
</body>
<!--base_header.htmlとbase_footer.htmlを読み込み、#headerと#footerに反映するjs-->
<script>
      fetch("base_header.html")//＊相対参照注意
        .then((response) => response.text())
        .then((data) => document.querySelector("#header").innerHTML = data);
      fetch("base_footer.html")//＊相対参照注意
        .then((response) => response.text())
        .then((data) => document.querySelector("#footer").innerHTML = data);
</script>
</html>

実はこれはこのサイトで使っているhtmlの一部です。
この例ではheaderとfooterとcssが共通化されています。

base_header.html・base_footer.htmlを読み込んで、.headerと.footerに代入するというJavaScriptが使われています。

また、cssはページ独自のものと、headerとfooterにつけるようの物の2つを用意しています。

今回のまとめ

  • 見出しはh1,h2,h3,h4,h5を使う
  • 横線はhr、終わりのタグはなし
  • 改行はbr、終わりのタグはなし
  • 段落はp
  • ハイパーリンクはa、リンク先は属性hrefに
  • グループ化はdiv、グループの名前は属性classに
  • htmlはざっくり、bodyとheadでできている
  • bodyはheaderとmainとfooterの3つで構成される
  • 綺麗なhtmlのコツは色のセレクト・余白の付け方
  • 無駄がないhtmlのコツは共通部分を外部ファイルに追い出すこと
  • メンテナウンス性を上げるにはコメント・グループ化が要
